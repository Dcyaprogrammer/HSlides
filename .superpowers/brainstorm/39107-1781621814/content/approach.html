Which architectural approach fits your vision?

Choose the foundation for your HTML slides platform

A

Simple JSON Data Recommended for MVP

Store slide metadata in data/slides.json. Hardcoded thumbnails, instant deployment, zero configuration.

Pros

  • Fastest to build (few hours)
  • Dead simple to edit
  • Perfect for 5-10 slides
  • Easy to export as static site

Cons

  • Manual thumbnail management
  • Need to rebuild for content changes
B

MDX Content Collections

Store each slide deck as an .mdx file with frontmatter. Type-safe, elegant content management.

Pros

  • Better for future growth
  • Easy to add blog/docs later
  • Git-friendly content editing

Cons

  • Slightly more complex setup
  • Overkill for 5-10 slides
C

Hybrid GitHub Integration

Fetch slide data dynamically from GitHub repos. Automated content discovery.

Pros

  • Automated content updates
  • Direct repo links
  • Scales to hundreds of slides

Cons

  • Requires GitHub API setup
  • Rate limiting concerns
  • Much more complex

My Recommendation: Option A (Simple JSON)

For a 5-10 slide prototype, JSON is perfect. You can iterate instantly, swap thumbnails easily, and migrate to MDX later if needed. Keep it simple first.
