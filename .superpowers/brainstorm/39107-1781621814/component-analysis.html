Neo-Brutalism UI Library + shadcn/ui Compatibility

How to combine these two libraries effectively

Good News: They're Compatible ✅

Both libraries use the same approach: copy-paste components (not npm packages). This means no dependency conflicts.

Neo-Brutalism UI Library

✓ React + Tailwind CSS

✓ Copy-paste components

✓ Custom brutalist colors

✓ Bold borders, hard shadows

! ~20 components

! No accessibility primitives

shadcn/ui

✓ React + Tailwind CSS

✓ Copy-paste components

✓ Built on Radix UI

✓ Full accessibility (ARIA)

✓ 50+ components

! Default styling (not brutalist)

Recommended Approach: Use Both Strategically

Base: Neo-Brutalism UI Library — For the visual aesthetic

  • Buttons, Cards, Inputs with brutalist styling
  • Custom color palette (pink, cyan, lime, violet, orange)
  • Bold borders and hard shadows built-in

Supplement: shadcn/ui — For complex interactive components

  • Dialogs, Dropdowns, Command palette (when needed)
  • Sheet (side panel), Toast (notifications)
  • Apply brutalist styling to shadcn components manually

Setup Strategy

Option 1: Start with Neo-Brutalism, add shadcn as needed

  • Install neo-brutalism-ui-library components first
  • Add shadcn/ui CLI for missing components
  • Override shadcn styles with brutalist CSS

Option 2: Start with shadcn/ui, apply brutalist theme

  • Install shadcn/ui normally
  • Copy neo-brutalism colors to tailwind.config.js
  • Apply borders/shadows globally
  • More flexible, more manual work

My Recommendation: Option 1

Start with neo-brutalism-ui-library for instant aesthetic. Add shadcn components only when you need something specific (like a dialog or complex dropdown). Faster to get the look you want.

Trade-off

You'll need to manually style shadcn components to match the brutalist aesthetic. But you'd have to do that anyway with Option 2 — just more upfront work.
